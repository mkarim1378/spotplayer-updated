## فیچرهای جدید

### مدیریت لایسنس

6. **لغو خودکار لایسنس پس از استرداد**: وقتی وضعیت سفارش به `refunded` تغییر می‌کنه، لایسنس رو به‌صورت خودکار غیرفعال کن (صفر کردن device ها).
   > **تأثیر بر سرعت:** بدون تأثیر روی کاربران عادی یا فرانت‌اند. فقط لحظه‌ای که ادمین سفارش رو مسترد می‌کنه یه API call اضافه می‌شه که چند صدم ثانیه طول می‌کشه. برای checkout و صفحات عمومی سایت کاملاً بی‌تأثیره.

7. **تمدید لایسنس**: دکمه‌ای در صفحه سفارش ادمین که تاریخ انقضای لایسنس رو از طریق API تمدید کنه.
   > **تأثیر بر سرعت:** خنثی در همه حالت‌ها. یه API call on-demand هنگام کلیک دکمه در ادمین. کاربران فرانت‌اند اصلاً درگیر نمی‌شند.

8. **نمایش تعداد دستگاه‌های فعال**: در باکس سفارش ادمین تعداد دستگاه‌هایی که لایسنس را فعال کرده‌اند نشان بده.
   > **تأثیر بر سرعت:** تأثیر منفی قابل‌توجه روی صفحه ویرایش سفارش در ادمین (اگه بدون caching پیاده بشه). برای هر بار باز شدن صفحه سفارش، یه API call به اسپات پلیر زده می‌شه که ۳۰۰–۸۰۰ms تأخیر ایجاد می‌کنه. راه‌حل: باید با AJAX lazy load یا transient با TTL کوتاه (~۵ دقیقه) پیاده بشه. برای فرانت‌اند و کاربران عادی بی‌تأثیره.

9. **Webhook از اسپات پلیر**: یه endpoint در وردپرس بساز که رویدادهای اسپات پلیر (مثل فعال‌سازی دستگاه) رو دریافت و در متادیتای سفارش ذخیره کنه.
   > **تأثیر بر سرعت:** خنثی برای کاربران سایت. فقط وقتی اسپات پلیر webhook می‌فرسته یه WordPress bootstrap اجرا می‌شه که روی سرور باره، نه روی تجربه کاربر. یه DB write ساده هم انجام می‌شه.

### تجربه کاربر

10. **ارسال کلید لایسنس در ایمیل**: پس از ساخت لایسنس، کلید رو در ایمیل تأییدیه سفارش ووکامرس یا یه ایمیل جداگانه بفرست.
    > **تأثیر بر سرعت:** اگه با WP Cron یا در پس‌زمینه انجام بشه: خنثی. اگه sync باشه: می‌تونه چند صدم ثانیه به زمان پردازش سفارش اضافه کنه، ولی کاربر این رو احساس نمی‌کنه چون صفحه تأیید سفارش قبلاً لود شده. برای فرانت‌اند عادی بی‌تأثیره.

11. **نمایش تاریخ انقضا به کاربر**: اگه API تاریخ انقضا برمی‌گردونه، در صفحه دوره‌های کاربر نمایش بده.
    > **تأثیر بر سرعت:** کاملاً بستگی به پیاده‌سازی داره. اگه تاریخ انقضا را در متادیتا ذخیره کنیم (هنگام ساخت لایسنس): خنثی، فقط یه DB read ساده‌ست. اگه برای نمایش API call بزنیم: تأثیر منفی جدی روی صفحه دوره‌های کاربر برای همه کاربران (۳۰۰–۸۰۰ms به ازای هر لایسنس). پیاده‌سازی صحیح: ذخیره local.

12. **غیرفعال‌سازی دستگاه توسط کاربر**: اگه API امکانش رو داشته باشه، کاربر بتونه از حساب کاربری‌اش یه دستگاه رو غیرفعال کنه.
    > **تأثیر بر سرعت:** خنثی روی صفحات عادی. فقط لحظه کلیک کاربر یه API call اجرا می‌شه. هیچ باری روی فرانت‌اند منفعلانه نداره.

### یکپارچه‌سازی

23. **WooCommerce Subscriptions**: وقتی اشتراک تجدید می‌شه، لایسنس رو تمدید کن؛ وقتی لغو می‌شه، غیرفعالش کن.
    > **تأثیر بر سرعت:** بدون تأثیر روی کاربران. هر دو عملیات (تمدید و لغو) توسط WP Cron در پس‌زمینه اجرا می‌شند. یه API call اضافه در جریان پردازش renewal. فرانت‌اند اصلاً درگیر نمی‌شه.

24. **WooCommerce Memberships**: دسترسی به لایسنس رو به عضویت گره بزن.
    > **تأثیر بر سرعت:** تأثیر منفی جزئی روی فرانت‌اند. هر بار که کاربر صفحه دوره‌ها رو باز می‌کنه یه چک عضویت اضافه می‌شه. اگه Memberships API کند باشه این تأثیر محسوس‌تره. می‌تونه ۲۰–۱۰۰ms اضافه کنه به صفحه دوره‌های کاربر.

25. **WP-CLI**: کامندهایی مثل `wp spotplayer create-license <order_id>` و `wp spotplayer disable-license <license_id>` برای سناریوهای اتوماسیون.
    > **تأثیر بر سرعت:** صفر. CLI فقط از خط فرمان اجرا می‌شه و هیچ overhead ای روی وب‌سایت نداره. در واقع می‌تونه به عنوان ابزار مدیریت offline بدون بار روی وب سرور استفاده بشه.

26. **REST API**: یه endpoint مثل `GET /wp-json/spotplayer/v1/license/<order_id>` برای ادغام با سیستم‌های خارجی.
    > **تأثیر بر سرعت:** تأثیر منفی بسیار جزئی. ثبت route جدید چند میکروثانیه به bootstrap وردپرس اضافه می‌کنه. هنگام فراخوانی endpoint یه WP bootstrap کامل + DB query اجرا می‌شه. برای فرانت‌اند و صفحات عادی کاملاً بی‌تأثیره.

## بهبود و بهینه‌سازی

### کد و معماری

27. **جایگزینی `eval()`**: تابع `spot_woo_license_data_eval` کد PHP رو با `eval()` اجرا می‌کنه. می‌شه از یه closure یا parser محدود استفاده کرد، یا حداقل خروجی رو بیشتر validate کرد.
    > **تأثیر بر سرعت:** تأثیر مثبت جزئی در همه جایی که لایسنس ساخته می‌شه. `eval()` نمی‌تونه توسط PHP OPcache کمپایل و کش بشه، یعنی هر بار مجدداً parse می‌شه. جایگزینی با closure کد رو OPcache-friendly می‌کنه.

28. **کشینگ با Transient**: نتیجه `spot_request_license_get` رو برای مدت کوتاهی (مثلاً ۵ دقیقه) در transient ذخیره کن تا تعداد درخواست‌های API کم بشه.
    > **تأثیر بر سرعت:** تأثیر مثبت قابل‌توجه روی پنل ادمین. هر بار که صفحه سفارش باز می‌شه به جای یه API call خارجی (۳۰۰–۸۰۰ms) از cache می‌خونه (زیر ۵ms).

29. **Retry خودکار**: اگه ساخت لایسنس با خطای شبکه شکست خورد (نه خطای منطقی)، یه بار دیگه با تأخیر کوتاه امتحان کن.
    > **تأثیر بر سرعت:** در حالت عادی (بدون خطا): خنثی. در صورت خطای شبکه: تأخیر ۱–۳ ثانیه به پردازش سفارش اضافه می‌کنه.

### 🔁 بازیابی خودکار لایسنس‌های ناموفق

37. **Cron بازیابی لایسنس‌های جا افتاده**: یه WP Cron هر ۱ ساعت اجرا بشه و سفارشاتی که بیش از ۱۵ دقیقه از تکمیلشون گذشته ولی هنوز `_spotplayer_data` ندارند رو پیدا کنه و مجدداً تلاش کنه لایسنس بسازه. این مستقیم‌ترین راه‌حل برای سفارشاتیه که به خاطر قطعی اینترنت یا خطای لحظه‌ای بی‌لایسنس موندند.
    > **کجا تأثیر داره:** پس‌زمینه سرور، هیچ تأثیری روی UX نداره. درصد موفقیت لایسنس‌ها را به نزدیک ۱۰۰٪ می‌رسونه.

### 📲 ارتباط با کاربر

38. **اطلاع‌رسانی SMS پس از ساخت لایسنس**: پس از ایجاد موفق لایسنس، یه پیامک حاوی کلید لایسنس برای کاربر ارسال کن. از افزونه‌های SMS متداول ایران (Digits SMS، Melipayamak، Kavenegar hook) پشتیبانی کن. این مهم‌ترین کانال ارتباطی در بازار ایرانه.
    > **کجا تأثیر داره:** فقط لحظه ساخت لایسنس، async. هیچ تأثیری روی UX سایت نداره.

39. **اعلان ادمین هنگام خطای لایسنس**: الان خطاها فقط توی notice ادمین نشون داده می‌شند که اگه ادمین اون لحظه در پنل نباشه می‌بینه. باید یه ایمیل هم به ادمین سایت فرستاده بشه که «لایسنس سفارش #X ساخته نشد» تا هیچ خریدی بی‌پاسخ نمونه.
    > **کجا تأثیر داره:** فقط در موارد خطا، async. ریسک از دست دادن خرید مشتری رو کاهش می‌ده.

### 🎁 قابلیت هدیه دادن دوره

40. **Gift License — خرید به نام دیگری**: در صفحه محصول یا checkout، گزینه‌ای برای «خرید برای دیگری» اضافه کن که کاربر بتونه نام و شماره گیرنده رو وارد کنه. لایسنس با مشخصات گیرنده (نه خریدار) ساخته می‌شه. این یه differentiator بزرگه که هیچ افزونه اسپات پلیری فعلاً نداره.
    > **کجا تأثیر داره:** checkout (یه فیلد اضافی) + منطق ساخت لایسنس. تأثیر سرعتی خنثی.

### 🔗 بهبود باکس ادمین سفارش

41. **لینک مستقیم به پنل اسپات پلیر**: در باکس سفارش ادمین، کنار شناسه لایسنس یه دکمه «مشاهده در پنل» بذار که به `https://panel.spotplayer.ir/license/edit/{id}` لینک بده. الان شناسه هست ولی باید copy کنی و بری پنل.

42. **Bulk Action در لیست سفارشات**: «ساخت لایسنس اسپات پلیر» رو به عنوان یه Bulk Action به لیست سفارشات ووکامرس اضافه کن. ادمین می‌تونه چند سفارش رو tick کنه و یه‌جا لایسنسشون رو بسازه بدون اینکه وارد هر سفارش بشه.
    > **کجا تأثیر داره:** فقط در لیست سفارشات ادمین هنگام استفاده.

43. **disable/enable لایسنس مستقیم از سفارش**: دکمه‌ای در باکس سفارش ادمین که بدون رفتن به پنل اسپات پلیر، لایسنس رو مستقیماً غیرفعال یا فعال کنه (صفر/بازیابی device count).

44. **Import/Export تنظیمات**: دکمه «دانلود تنظیمات» که تنظیمات افزونه رو به JSON export کنه و دکمه «بارگذاری تنظیمات» برای import. خیلی مفیده برای migration بین سرورها یا backup قبل از تغییر کلید API.

### 🎓 داشبورد یادگیرنده (Learner Dashboard)

45. **جایگزین شورت‌کد با صفحه کامل‌تر**: شورت‌کد فعلی فقط لیست دوره‌ها رو نشون می‌ده. یه صفحه کامل‌تر بساز که:
    - کارت هر دوره با تصویر بندانگشتی محصول ووکامرس
    - نشان‌دهنده تاریخ خرید
    - badge وضعیت لایسنس (فعال / منقضی)
    - دکمه «شروع یادگیری» که مستقیم پلیر وب رو باز می‌کنه (اگه web فعال باشه)

46. **جستجو در دوره‌های کاربر**: یه فیلد جستجو روی شورت‌کد که کاربر بتونه بین دوره‌هایش سرچ کنه. برای کاربرانی که ۱۰+ دوره دارند خیلی مفیده.

47. **فیلتر وضعیت دوره**: در صفحه دوره‌های کاربر، tab هایی مثل «همه / فعال / منقضی» اضافه کن. نیاز به ذخیره تاریخ انقضا در local metadata داره.

### ⚡ بهبودهای فنی کوچک

62. **اضافه کردن `noindex` به صفحه pending**: وقتی کاربر صفحه تأیید سفارش رو با پیام «در حال آماده‌سازی» می‌بینه، این صفحه نباید توسط ربات‌های موتور جستجو ایندکس بشه. یه meta tag `noindex` به حالت pending اضافه کن.

63. **محافظت از `/spdeb` با IP whitelist**: endpoint دیباگ `/spdeb` الان فقط با بررسی `administrator` role محافظت می‌شه. محدود کردنش به IP ادمین سرور امنیت رو بالاتر می‌بره.

64. **پشتیبانی از محصولات Grouped در ووکامرس**: الان فقط Simple و Variable پشتیبانی می‌شند. اگه محصولات Grouped استفاده بشه (چند دوره در یه package) باید هر آیتم داخل group جداگانه بررسی بشه.

65. **Auto-detection نسخه WooCommerce HPOS**: افزونه باید در activation چک کنه که آیا HPOS فعاله یا نه و در صورت نیاز تابع مناسب رو صدا بزنه، به جای اینکه با `get_edit_post_link` شکست بخوره.

66. **کشینگ نتیجه `wc_get_products` در صفحه تنظیمات**: در صفحه تنظیمات، فرم ایجاد سفارش دسته‌ای `wc_get_products(['limit' => -1])` اجرا می‌کنه که روی سایت‌های با محصولات زیاد کند می‌شه. این رو باید با transient کش کرد.

### 📦 باکس اسپات پلیر در صفحه ویرایش سفارش — ایده‌های تخصصی

> باکس فعلی فقط یه فرم ساده با شناسه، نام، ۳ واترمارک، و دو دکمه داره. اینجا هر کاری که می‌شه روی این باکس کرد رو آوردم.

**وضعیت و اطلاعات لایسنس:**

67. **badge وضعیت لایسنس**: بالای باکس یه badge رنگی نشون بده — 🟢 فعال / 🔴 غیرفعال / 🟡 منقضی / ⚫ بدون لایسنس. این اطلاعات رو هنگام باز کردن صفحه از API (با AJAX lazy load + transient ۵ دقیقه) بکش تا صفحه بلاک نشه.

68. **نمایش تعداد دستگاه‌های فعال / کل ظرفیت**: مثلاً «۲ از ۳ دستگاه فعال». با همون AJAX lazy load از API بکش. اگه ظرفیت پر باشه badge قرمز بشه.

69. **نمایش تاریخ ایجاد لایسنس**: تاریخ ساخت لایسنس رو که در `_spotplayer_data` ذخیره‌ست به شمسی نمایش بده. الان این اطلاعات وجود داره ولی نشون داده نمی‌شه.

70. **نمایش تاریخ انقضا با هشدار رنگی**: اگه تاریخ انقضا در `_spotplayer_data` باشه، نشونش بده. اگه کمتر از ۷ روز مونده باشه قرمز بشه، اگه کمتر از ۳۰ روز نارنجی.

71. **نمایش نام دوره‌ها به جای شناسه**: الان در باکس شناسه hex دوره نشون داده می‌شه. اگه نام دوره رو از متادیتای محصول ووکامرس بکشیم (که `_spotplayer_course` داره) نام فارسی دوره رو نشون بده.

72. **نمایش کلید لایسنس با دکمه کپی**: کلید لایسنس (`_spotplayer_data['key']`) الان اصلاً در باکس ادمین نمایش داده نمی‌شه. یه ردیف اضافه کن که کلید رو (به صورت masked: `XXXX...XXXX`) نشون بده با دکمه کپی و یه آیکون نمایش/پنهان.

**اکشن‌های مستقیم روی لایسنس:**

73. **دکمه غیرفعال‌سازی لایسنس (ریست دستگاه‌ها)**: یه دکمه AJAX که device count رو صفر کنه — بدون نیاز به رفتن به پنل اسپات پلیر. با یه confirm dialog «آیا مطمئنید؟» همراه باشه.

74. **دکمه فعال‌سازی مجدد**: اگه لایسنس غیرفعاله (device=0)، یه دکمه «فعال‌سازی مجدد» که device limit رو به مقدار اولیه برگردونه.

75. **تغییر محدودیت دستگاه از باکس**: یه input عددی کوچک که بشه مستقیماً از همین باکس device limit رو تغییر داد و با یه AJAX call به API ارسال کرد.

76. **تمدید تاریخ انقضا از باکس**: یه input date یا یه dropdown (مثلاً: «+۳۰ روز / +۶۰ روز / +۱ سال / تاریخ دلخواه») که تاریخ انقضا رو از همین باکس تمدید کنه.

77. **تغییر واترمارک روی لایسنس موجود**: الان وقتی لایسنس وجود داره فیلدهای واترمارک disabled هستند. یه گزینه «ویرایش واترمارک» اضافه کن که فیلدها رو فعال کنه و با یه AJAX call مستقیماً واترمارک لایسنس موجود رو در اسپات پلیر آپدیت کنه — بدون ساخت لایسنس جدید.

78. **تغییر محدودیت سرفصل‌ها (Limit) از باکس**: مثلاً `0-` یا `1,4-6`. این الان فقط در bulk و از طریق `_spot_limit` در سفارش قابل تنظیمه. باید از همین باکس هم قابل ویرایش باشه.

79. **ارسال مجدد ایمیل لایسنس به مشتری**: دکمه‌ای که ایمیلی حاوی کلید لایسنس رو مجدداً به ایمیل مشتری ارسال کنه. برای وقتی که مشتری ادعا می‌کنه «ایمیل نگرفتم».

80. **ارسال SMS لایسنس**: مشابه ۷۹ ولی از طریق پلاگین SMS فعال سایت.

81. **Regenerate لایسنس**: دکمه «ساخت مجدد لایسنس» که لایسنس فعلی رو حذف (یا نادیده) بگیره و با همون مشخصات یه لایسنس جدید بسازه. مفیده وقتی لایسنس خراب شده.

82. **انتقال لایسنس به سفارش دیگر**: فیلدی که شناسه سفارش مقصد رو بگیره و `_spotplayer_data` رو از این سفارش به اون منتقل کنه (با confirm). برای وقتی که مشتری غلطی سفارش داده.

**طراحی و UX:**

83. **تب‌بندی باکس**: باکس رو به چند تب تقسیم کن — «اطلاعات» (همه چیزی که الان هست)، «دستگاه‌ها» (لیست AJAX دستگاه‌های فعال اگه API بده)، «محدودیت‌ها» (device limit, chapter limit, expiry)، «اکشن‌ها» (disable/enable/resend). این UX رو خیلی تمیزتر می‌کنه.

84. **Collapse/Expand باکس**: اگه لایسنس وجود داره، باکس رو به صورت collapsed نشون بده (فقط badge وضعیت و دکمه «باز کردن») تا صفحه ویرایش سفارش شلوغ نشه.

85. **نمایش لاگ تغییرات داخل باکس**: آخرین ۵ تغییری که از طریق افزونه روی این لایسنس انجام شده (ساخت، غیرفعال‌سازی، ارسال ایمیل) رو با timestamp نشون بده.

86. **هشدار برای سفارش‌های بدون لایسنس**: اگه سفارش محصول اسپات پلیر داره ولی لایسنس نداره، بالای باکس یه notice زرد رنگ نشون بده: «این سفارش هنوز لایسنس ندارد. دلیل: [آخرین خطا از order notes]».

87. **متن خوانا برای کد خطا**: الان کدهای خطا عدد هستند. یه mapping از کدهای رایج اسپات پلیر به پیام‌های فارسی فهمیدنی ایجاد کن.

### 🎓 شورت‌کد [spotplayer_courses] — ایده‌های تخصصی

> شورت‌کد فعلی: لیست دوره‌ها رو ۱۲ تایی نشون می‌ده، صفحه‌بندی داره، با کلیک پلیر/دانلود باز می‌شه. اینجا هر چیزی که می‌شه بهش اضافه کرد رو آوردم.

**ظاهر کارت‌های دوره:**

88. **تصویر شاخص محصول**: الان `$p->get_image()` تصویر محصول ووکامرس رو می‌کشه ولی size استاندارد نیست. از `get_the_post_thumbnail($product_id, 'medium')` استفاده کن و اگه تصویر نبود یه placeholder نشون بده.

89. **نام دوره به عنوان عنوان کارت**: الان نام محصول ووکامرس نشون داده می‌شه. گزینه‌ای اضافه کن که بتونه نام دوره رو از API اسپات پلیر (اگه cache باشه) بکشه — برای وقتی که نام دوره در اسپات پلیر با نام محصول ووکامرس فرق داره.

90. **تاریخ خرید زیر کارت**: تاریخ خرید (`$o->get_date_created()`) رو به شمسی نمایش بده. برای کاربر شفافیت ایجاد می‌کنه.

91. **badge وضعیت روی کارت**: گوشه کارت یه badge نشون بده: 🟢 فعال / 🔴 منقضی / 🕐 در حال آماده‌سازی. این نیاز به ذخیره وضعیت در local metadata داره.

92. **نوار پیشرفت تماشا (Progress Bar)**: اگه API اسپات پلیر درصد تماشا رو برگردونه، یه progress bar در پایین کارت نشون بده. حتی اگه API الان این اطلاعات رو نمی‌ده، ساختار رو آماده کن.

93. **نمایش تعداد سرفصل‌ها**: اگه نام دوره از API کشیده می‌شه، تعداد `items` هم در دسترسه — «۱۲ سرفصل» زیر عنوان نشون بده.

94. **تایمر انقضا**: اگه تاریخ انقضا در متادیتا ذخیره باشه (ایده ۴۸)، زیر کارت یه متن نشون بده: «۱۵ روز مانده» و اگه کمتر از ۷ روز مونده رنگش قرمز بشه.

95. **badge «جدید»**: اگه سفارش کمتر از ۷ روز پیش ثبت شده، یه badge «جدید» روی کارت نشون بده.

96. **دکمه کپی مستقیم کلید**: یه آیکون copy روی خود کارت دوره (tooltip: «کپی کلید لایسنس») بذار که کاربر بدون رفتن به صفحه داخلی بتونه کلیدش رو کپی کنه.

**ناوبری و فیلتر:**

97. **جستجوی live**: یه input جستجو بالای لیست که با JavaScript (بدون reload) کارت‌ها رو فیلتر کنه بر اساس نام دوره.

98. **tab های وضعیت**: سه tab — «همه» / «فعال» / «منقضی» — که بر اساس وضعیت ذخیره‌شده در metadata فیلتر کنن.

99. **مرتب‌سازی**: یه dropdown با گزینه‌های «جدیدترین / قدیمی‌ترین / الفبایی». مرتب‌سازی client-side روی کارت‌های لود شده باشه (بدون reload).

100. **دسته‌بندی دوره‌ها**: اگه محصولات ووکامرس دسته‌بندی دارند، دوره‌ها رو بر اساس category گروه‌بندی کن و هر گروه یه header داشته باشه.

**چیدمان و طراحی:**

101. **سوئیچ Grid/List**: یه toggle بالای صفحه که بین نمایش grid (3 ستونه) و list (یه ستون با اطلاعات بیشتر) سوئیچ کنه. انتخاب کاربر در localStorage ذخیره بشه.

102. **Responsive بهتر**: الان روی موبایل هنوز ۳ ستون نشون می‌ده. با CSS media query به ۱ ستون روی موبایل و ۲ ستون روی تبلت تبدیل کن.

103. **انیمیشن ورود کارت‌ها**: با CSS animation ساده، کارت‌ها هنگام لود به ترتیب fade-in بشند. سبک و بدون JavaScript.

104. **حالت Empty State**: وقتی کاربر هیچ دوره‌ای نداره، یه پیام دوستانه با آیکون و لینک «مشاهده دوره‌ها» نشون بده به جای خالی ماندن صفحه.

105. **شخصی‌سازی رنگ کارت‌ها**: به هر محصول ووکامرس یه رنگ متادیتا بده (`_spot_card_color`) که کارتش رو با اون رنگ نشون بده — مثلاً gradient header.

**امکانات پیشرفته:**

109. **پیام خوش‌آمد شخصی**: بالای لیست یه متن مثل «خوش اومدی محمد، ۳ دوره داری» نشون بده. از `wp_get_current_user()` بکش.

110. **شمارنده دوره‌ها**: کنار عنوان صفحه یه chip نشون بده: «۵ دوره». کاربر یه‌نگاهه می‌فهمه چقدر دوره داره.

112. **قفل نمایش دوره‌های منقضی**: به جای حذف دوره‌های منقضی از لیست، اون‌ها رو با overlay تاریک و آیکون 🔒 نشون بده و یه لینک «تمدید دوره» روشون بذار.

113. **اشتراک‌گذاری دوره**: یه آیکون share روی هر کارت که با Web Share API (موبایل) یا کپی لینک (دسکتاپ) عمل کنه.

114. **علاقه‌مندی (Bookmark)**: یه آیکون ستاره روی کارت که وضعیتش در `user_meta` ذخیره بشه. کاربر بتونه دوره‌های مورد علاقه‌اش رو star کنه و فیلتر «علاقه‌مندی‌ها» داشته باشیم.

115. **تاریخچه آخرین مشاهده**: زیر نام دوره متن «آخرین بازدید: ۳ روز پیش» نشون بده. این اطلاعات رو با JavaScript در `localStorage` ذخیره کن (هر بار که کاربر وارد صفحه دوره می‌شه timestamp ذخیره بشه).

116. **نمایش تعداد سرفصل باز و قفل**: اگه محدودیت سرفصل (`limit`) روی لایسنس اعمال شده، نشون بده چند سرفصل در دسترس و چند تا قفله. مثلاً: «۴ سرفصل آزاد / ۸ سرفصل قفل».

**پارامترهای شورت‌کد:**

117. **`limit`**: `[spotplayer_courses limit="6"]` — فقط ۶ دوره نشون بده (بدون pagination).
118. **`product`**: `[spotplayer_courses product="123,456"]` — فقط دوره‌های مرتبط با این محصولات.
119. **`layout`**: `[spotplayer_courses layout="grid|list|compact"]` — نوع چیدمان.
120. **`show_expired`**: `[spotplayer_courses show_expired="false"]` — پنهان کردن منقضی‌ها.
121. **`order`**: `[spotplayer_courses order="date_desc|date_asc|name_asc"]` — ترتیب پیش‌فرض.
122. **`title`**: `[spotplayer_courses title="دوره‌های من"]` — عنوان قابل تنظیم بالای لیست.
123. **`cta_text` و `cta_url`**: `[spotplayer_courses cta_text="خرید دوره جدید" cta_url="/shop"]` — دکمه CTA سفارشی.
124. **`columns`**: `[spotplayer_courses columns="2"]` — تعداد ستون‌های grid.
