# برنامه‌ریزی: ارسال پیامک کلید لایسنس

---

## هدف کلی

بعد از ایجاد موفق لایسنس برای هر سفارش، کلید لایسنس رو از طریق پیامک به شماره موبایل خریدار ارسال کنیم — بدون اینکه مشتری مجبور باشه وارد سایت بشه تا لایسنسش رو پیدا کنه.

---

## چه موقع پیامک ارسال بشه؟

| رویداد | توضیح | اولویت |
|---|---|---|
| ساخت خودکار لایسنس پس از تکمیل سفارش | رایج‌ترین حالت | بالا |
| ساخت لایسنس در پس‌زمینه (Async) | بعد از اجرای action در Action Scheduler | بالا |
| ساخت دستی لایسنس توسط ادمین از صفحه سفارش | دکمه «ایجاد لایسنس» | متوسط |
| ارسال مجدد دستی توسط ادمین | دکمه جداگانه «ارسال پیامک» در صفحه سفارش | متوسط |
| bulk license creation از CSV | اختیاری — شاید نخواد همه رو SMS بزنه | پایین |

---

## منابع شماره موبایل

اولویت خواندن شماره (از بالا به پایین):

1. `$order->get_billing_phone()` — شماره وارد‌شده در فرم پرداخت
2. `get_user_meta($uid, 'digits_phone', true)` — شماره تأیید‌شده از افزونه Digits
3. `get_user_meta($uid, 'digits_phone_no', true)` — فیلد جایگزین Digits
4. `get_user_meta($uid, 'billing_phone', true)` — فیلد متا کاربر

**نرمال‌سازی شماره:**
- حذف همه کاراکترهای غیرعددی
- اگه با `0` شروع میشه: تبدیل به `98XXXXXXXXXX`
- اگه با `98` شروع میشه: همون رو نگه دار
- اگه ۱۰ رقمه (بدون پیش‌شماره): `98` جلوش بذار

## پروایدرهای پیامک

### پروایدرهای ایرانی اولویت‌دار

| پرواید | API | محبوبیت | قیمت | داکیومنتیشن |
|---|---|---|---|---|
| **کاوه‌نگار (Kavenegar)** | REST + SDK | بسیار بالا | متوسط | عالی |
| **ملی‌پیام (Melipayamak)** | REST + SOAP | بالا | پایین | خوب |
| **فراپیامک** | REST | متوسط | پایین | متوسط |
| **گرین‌وب (GreenWeb)** | REST | متوسط | پایین | خوب |
| **سینا پیامک** | REST | پایین | پایین | متوسط |
| **اسمس‌ایر (SMSIR)** | REST | متوسط | متوسط | خوب |

### رویکرد معماری

به جای hardcode کردن یه پروایدر خاص، یه **لایه انتزاعی (Adapter Pattern)** طراحی کنیم:

```
includes/sms/
├── interface.php          ← قرارداد کلی برای همه پروایدرها
├── send.php               ← تابع اصلی ارسال با منطق retry و logging
├── providers/
│   ├── kavenegar.php
│   ├── melipayamak.php
│   ├── farapayamak.php
│   └── custom.php         ← پروایدر سفارشی با URL و پارامترهای قابل تنظیم
```

### Interface پروایدر

```php
interface SpotSmsProvider {
    // $to: شماره با فرمت بین‌المللی (98XXXXXXXXXX)
    // $message: متن پیام نهایی
    // throws Exception در صورت خطا
    public function send(string $to, string $message): void;
}
```

### پروایدر Custom (مهم‌ترین)

کاربران زیادی پروایدرهای کمتر معروف دارند. یه پروایدر «سفارشی» بسازیم که:
- URL endpoint قابل تنظیم
- نوع درخواست: GET یا POST
- پارامترهای قابل تعریف با placeholder:
  - `{phone}` → شماره گیرنده
  - `{message}` → متن پیامک
  - `{apikey}` → کلید API
- مثال: `https://api.example.com/send?receptor={phone}&message={message}&apikey={apikey}`

## متن پیامک (Template)

### متغیرهای در دسترس برای template

| متغیر | مقدار | مثال |
|---|---|---|
| `{key}` | کلید لایسنس کامل | `ABCD1234...` |
| `{name}` | نام مشتری | `علی احمدی` |
| `{order_id}` | شماره سفارش | `1042` |
| `{site_name}` | نام سایت | `آکادمی کارنو` |
| `{courses_url}` | لینک صفحه دوره‌ها | `https://...` |

### template پیش‌فرض

```
{site_name}
لایسنس دوره شما آماده‌ست!
کلید: {key}
برای مشاهده دوره‌ها: {courses_url}
```

### نکات template

- کاربر باید بتونه متن رو کاملاً سفارشی کنه
- اگه template خالی باشه → پیامک ارسال نشه
- طول پیام باید نمایش داده بشه (هر SMS معمولاً ۱۶۰ کاراکتر لاتین یا ۷۰ یونیکد)
- کلید لایسنس خیلی طولانیه (~۸۸ کاراکتر) → پیامک حتماً چندتکه میشه — باید به کاربر نشون داده بشه

## منطق ارسال (send.php)

```php
function spot_sms_send_license(int $order_id, string $key): void {
    // 1. بررسی فعال بودن SMS
    // 2. گرفتن پروایدر از تنظیمات
    // 3. ساختن پروایدر
    // 4. خواندن شماره موبایل از سفارش/کاربر
    // 5. نرمال‌سازی شماره
    // 6. render کردن template
    // 7. ارسال
    // 8. لاگ نتیجه در یادداشت سفارش
}
```

### Retry منطق

- اگه ارسال اول شکست خورد: یه بار دیگه با تأخیر ۶۰ ثانیه (با WP Cron)
- بیشتر از ۲ بار retry نزنیم
- خطاهای validation (شماره اشتباه، template خالی) → retry نزن

## نقاط hook در کد فعلی

### WooCommerce

در `includes/woocommerce/functions.php` داخل `spot_woo_order_license_request`:
```php
// بعد از:
$ord->update_meta_data('_spotplayer_data', $data = array_merge($data, $rep));
$ord->save_meta_data();

// اضافه بشه:
spot_sms_send_license($ord->get_id(), $data['key']);
```

### Background async

در `includes/woocommerce/functions.php` داخل `spot_run_license_async`:
```php
// بعد از موفقیت:
spot_sms_send_license($order_id, $result['key']);
```

### EDD

در `includes/edd/functions.php` داخل `spot_edd_payment_license_request`:
```php
spot_sms_send_license_edd($pay->ID, $data['key']);
```

## دکمه ارسال مجدد در ادمین

در `includes/admin/order-box.php` وقتی سفارش لایسنس داره:
```html
<button type="submit" name="spot-resend-sms" value="1">ارسال مجدد پیامک لایسنس</button>
```

در handler ادمین:
```php
if (!empty($_POST['spot-resend-sms'])) {
    $data = spot_woo_license_data($ord);
    if (!empty($data['key'])) {
        spot_sms_send_license($ord->get_id(), $data['key']);
        spot_admin_notice('پیامک لایسنس برای این سفارش ارسال شد.', 'success');
    }
}
```

## مدیریت خطا و لاگ

### یادداشت سفارش

موفقیت:
```
📱 پیامک لایسنس به شماره 09XXXXXXXXX ارسال شد.
```

خطا:
```
⚠️ ارسال پیامک لایسنس ناموفق بود: [پیام خطا]. شماره: 09XXXXXXXXX
```

شماره نامعتبر:
```
⚠️ پیامک ارسال نشد — شماره موبایل معتبر برای این سفارش یافت نشد.
```

## نکات امنیتی

- کلید API پیامک باید در DB ذخیره بشه (مثل کلید API اسپات‌پلیر) — نه در کد
- template باید از تزریق کد PHP مصون باشه → فقط جایگزینی string ساده، نه `eval`
- شماره موبایل در لاگ‌ها باید masking بشه: `0912***1234`
- لاگ کامل شماره فقط توی یادداشت سفارش که فقط ادمین می‌بینه

## سوالات باز / تصمیمات لازم

1. **چند پروایدر همزمان ساپورت کنیم یا یکی؟**
   → پیشنهاد: Kavenegar + Custom در فاز اول کافیه

2. **آیا bulk CSV هم باید SMS بزنه؟**
   → احتمالاً نه — اون عملیات داخلی‌تره

3. **Template جداگانه برای EDD و WooCommerce؟**
   → یه template واحد کافیه، فقط متغیرها فرق می‌کنن

4. **آیا تاریخچه SMS ارسال‌شده‌ها ذخیره بشه؟**
   → در یادداشت سفارش کافیه، جدول جداگانه لازم نیست

5. **اگه کاربر چند لایسنس در یه سفارش داشت چی؟**
   → یه پیامک با همه کلیدها؟ یا چند پیامک؟
   → پیشنهاد: یه پیامک با کلیدها به صورت لیست (جدا با خط جدید)

6. **WP Cron برای retry یا Action Scheduler؟**
   → اگه Action Scheduler موجوده از اون استفاده کنیم (مثل سیستم bulk موجود)
